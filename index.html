Fraction.less Boilerplate

What is it?

Fraction.less is a HTML5/CSS3 Boilerplate and Grid System for rapid layout. Fraction.less is well integrated with the LESS stylesheet language but you're free to choose regular CSS if you wish. The grid system is perfect for web design newbies as well as experienced pros, allowing anyone to quickly build deploy page layouts within minutes.

Why use it?

Do we really need yet another boilerplate? Eh, sure, why not? Fraction.less lets you develop sites quickly. It's different from other frameworks in that it attempts to cut down on the amount of boilerplate code it comes with and emphasizes modularity and choice. It's incredibly simple with almost no learning curve.

Custom workflows

Fraction.less incorporates the best parts of several development tools in one easy to use package. Use the tools you already know and love, there's no need to adapt to a new workflow, and benefit from the easiest, least convoluted, LESS-based grid system in the history of the entire universe. Fraction.less is everything you need without any extra bloat.

Who uses Fraction.less?

Typography

Fraction.less has basic typography presets based on the golden grid. Because Fraction.less' philosophy is to not lock the user into learning a framework these presets are minimal and are meant as a great jumping off point.

Buttons

Fraction.less comes with a starting point for creating beautiful buttons from regular anchor links. Get the most out of Fraction.less buttons by taking advantage of LESS.

Button Example

VanityJS

Get common jQuery effects like sliders, tabs, accordions, and more up and running in no time at all with built in VanityJS support.

Extend with LESS

Easily modify any styles using LESS or work in regular CSS if you prefer. Either way you'll never need to change your current workflow.

HTML5-based

Fraction.less is largely based on HTML5 Boilerplate and shares the philosophy of trying not to tie you into a certain way of doing things. We added some good parts and removed some bloat.

jQuery baked in

We use jQuery and supply the 1.6.4 library in minified form.

1140px Grid

Fraction.less' grid is based on the 1140 Grid System.

Compatible

The Fraction.less Boilerplate with all its modular components is compatible with all major browsers including IE6!

How it works

You can think of the Fraction.less grid like an old fashioned HTML table. It consists of classes defining rows and cells. Each "row" is always the same width with each "cell" varying in size, giving the layout its basic structure. There's more to it than that, though. The Fraction.less grid is easy - just take a look at this sample code which produces the first half of this page.

The example HTML code to the right sets up a grid as shown below.

Example code

<header class="container">
	<section class="row">
    	<article class="one centered">
            <!-- Block of centered content -->
        </article>
    </section>
</header>
<section class="container">
    <section class="row">
        <article class="one-third">
            <!-- 1/3 width bucket -->
        </article>
	    <article class="one-third">
		    <!-- Bucket 2, Row 1 -->
	    </article>
	    <article class="one-third last">
		    <!-- Bucket 3, Row 1 -->
	    </article>
    <section>
</section>

Just add it up

The Fraction.less grid works using a few simple, easy to remember classes. If you can add simple fractions (or spare change) then you can have a page layout up in 5 minutes or less! Here's how it works: you create a container element that holds an unlimited number of rows and fill it with elements with class names that add up to 1. Classes in Fraction.less are named after fractions so it's easy! For example: .one-quarter + .one-half + .one-quarter = 1 = .full-width

The .container class is perfect for blocks of elements needing full-screen backgrounds and .rows are good for, well... creating rows. So be sure to wrap your fractions around a .container and .row (in that order). The last element in each row gets a .last class. That's all there is to it.
